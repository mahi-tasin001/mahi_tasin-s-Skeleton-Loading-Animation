# mahi_tasin-s-Skeleton-Loading-Animation
Skeleton Loading Animation

Tutorial:



Skeleton Loading Animation is used to improve the user experience while the cocntent loading period. Instead of loading a blank page or a simple loading spinner, the website shows temporary placeholder shapes that resemble the actual layout of the content. In this project, animated gray blocks appear where images, text and profile information will eventually be displayed. Once the content finishes loading, these placeholders disappear and the real content is shown.


#The project was developed with the use of following tools/languages:

1.HTML: For basic structure and content layout of the webpage.
2.CSS: Designing skeleton elements and adding animation effects.
JavaScript: Loading process and replaces placeholder elements witht the actual content after a small delay.

#Steps to build and run the project are :

 1. Designing the webpage layout using HTML is the first step. This consisnts creating sections for images, headings, txt and user profile information.
 2. Skeleton classes are added to the elements that should display loading placeholders. These placeholders occupy the same space as the actual content.
 3. CSS is used to give the placeholders a gray appearance and create a subtle animation that refers to the content is still loading.
 4. JavaScript is used to simulate the loading process. The skeleton classes are removed after a short delay and the real content comes visible.

Running the project:

 1. Save HTML, CSS, and JavaScript files.
 2. Open the HTML file in a web browser.
 3. The skeleton placeholders will appear immediately.
 4. After the leading period ends, the actual content will be displayed.

#Key Features & Challenges:

 1. Provides a smooth loading experience instead of showing blank screen.
 2. Makes the website feel faster and more interactive.
 3. Simple to build using HTML, CSS and JavaScript.

Challenges:

 1. Creating placeholders that match the actual content layout.
 2. Keeping the animation smooth without affecting performance.
 3. Making sure the design works properly on the different screen sizes.
